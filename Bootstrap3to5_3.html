Bootstrap 3系からBootstrap 5.3系への変換仕様

既存HTML/CSS/JavaScriptをBootstrap 5.3系へ移行するための変換ルール、代替コンポーネント、確認観点を整理した設計ドキュメント。

目次

1. 変換方針

重要 Bootstrap 3から5.3への移行は、class名置換だけでは完了しない。Bootstrap 4で大規模な再設計が行われ、Bootstrap 5でjQuery依存が削除され、5.3でCSS variablesとcolor modesが強化されているため、段階的に差分を吸収する。

観点Bootstrap 3系Bootstrap 5.3系変換仕様
対象バージョンBootstrap 3.3.x / 3.4.1Bootstrap 5.3.x3系のHTML/CSS/JS実装を5.3系の構文・コンポーネントに変換する。
移行方式jQuery + Bootstrap JSVanilla JS + Bootstrap bundlejQuery前提の初期化・イベント処理をBootstrap 5のJS APIへ変換する。
CSS設計Less、物理方向 left/right、古いutilitySass、CSS variables、start/end、utility API独自テーマはLess変数の単純置換ではなく再設計対象にする。
グリッド.col-xs-* など.col-* / .col-sm-* などxs接頭辞は廃止され、最小幅指定なしの .col-* に変換する。
コンポーネントpanel / well / thumbnail / glyphicon ありcard / utilities / Bootstrap Icons等削除・統合されたコンポーネントは代替設計に置き換える。
data属性data-toggle / data-target / data-dismissdata-bs-toggle / data-bs-target / data-bs-dismissすべてのBootstrap JS用data属性に bs 名前空間を付ける。
推奨手順は、①画面・共通部品の棚卸し、②class名の機械変換、③削除コンポーネントの設計置換、④JS初期化のVanilla JS化、⑤レスポンシブ・E2E確認、の順で進めること。

2. 依存関係・読み込みの変換仕様

対象Bootstrap 3系Bootstrap 5.3系変換仕様・注意点
CSS CDNbootstrap/3.x/css/bootstrap.min.cssbootstrap@5.3.x/dist/css/bootstrap.min.css旧CSSを削除し、5.3系CSSへ差し替える。
JS CDNjquery.js + bootstrap/3.x/js/bootstrap.min.jsbootstrap@5.3.x/dist/js/bootstrap.bundle.min.jsbundleにはPopperが含まれる。jQueryはBootstrap用途では不要。
jQuery必須不要自作処理でjQueryを使う場合は残せるが、Bootstrap plugin呼び出しはVanilla JSへ移行する。
PopperTooltip/Popoverで内部依存Popper v2系bundle版を使うと個別読み込み不要。
IconsGlyphicons同梱同梱なしBootstrap Icons、Font Awesome、社内アイコン等に置換する。
ブラウザIE対応を考慮する実装が多いIE非対応IE用polyfill、条件付きCSS、古いflexbox対策を棚卸しする。

読み込み例

<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/css/bootstrap.min.css" rel="stylesheet">

<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/js/bootstrap.bundle.min.js"></script>

3. グリッド・レイアウト変換仕様

Bootstrap 3系Bootstrap 5.3系変換仕様注意点
.col-xs-6.col-6xs接頭辞を削除する。最小ブレークポイントから有効。
.col-sm-6.col-sm-6原則そのまま。グリッド仕様差分による余白確認は必要。
.col-md-offset-2.offset-md-2offset構文に変換。xs相当は .offset-*。
.col-md-push-4 / .col-md-pull-8.order-md-*push/pullはorder utilityへ再設計。完全な機械変換は難しいため画面確認必須。
.container.container原則そのまま。最大幅が変わるため画面幅ごとの確認が必要。
.container-fluid.container-fluid原則そのまま。余白・ネストを確認。
.row.row原則そのまま。Bootstrap 5のgutterはCSS variablesで制御される。
.clearfix visible-xs-block.clearfix d-block d-sm-none表示utilityに変換。clearfix自体の必要性も見直す。

4. Utility class 変換仕様

Bootstrap 3系Bootstrap 5.3系変換仕様
.pull-left.float-startRTL対応のstart/end命名へ変換。
.pull-right.float-endRTL対応のstart/end命名へ変換。
.center-block.mx-auto d-blockblock化と左右auto marginで中央寄せ。
.hidden-xs.d-none d-sm-blockxsのみ非表示。要件に応じてdisplay種類を調整。
.visible-xs-block.d-block d-sm-nonexsのみ表示。inline/table等は用途別に変換。
.text-left.text-start方向指定をstart/endへ。
.text-right.text-end方向指定をstart/endへ。
.text-muted.text-body-secondary または .text-muted5.3の色設計に合わせて選択。既存互換ならtext-mutedでも可。
.sr-only.visually-hiddenスクリーンリーダー用class名を変更。
.sr-only-focusable.visually-hidden-focusableフォーカス時表示用class名を変更。

6. フォーム変換仕様

Bootstrap 3系Bootstrap 5.3系変換仕様
.form-group.mb-3フォーム部品の余白はspacing utilityへ。
.control-label.form-labellabelに明示。
.form-control.form-control原則そのまま。
.input-lg.form-control-lgサイズ指定を変更。
.input-sm.form-control-smサイズ指定を変更。
.help-block.form-text補足テキストへ。
.form-control-static.form-control-plaintext読み取り専用風テキストへ。
.checkbox / .radio.form-checkinputとlabel構造を5系に合わせる。
.checkbox-inline / .radio-inline.form-check form-check-inlineinline表示へ。
.has-error.is-invalid + .invalid-feedback親要素状態から入力要素状態へ変更。
.has-success.is-valid + .valid-feedback検証状態を入力要素に付与。
.input-group-addon.input-group-text入力グループのテキスト要素へ。
.input-group-btn直接 .btn を配置prepend/append用wrapperは不要。
.form-inlinegrid / flex / spacing utility専用classは廃止。レイアウトとして再設計。
Bootstrap 5ではフォームの専用レイアウトclassが減り、grid / flex / spacing utilityで組む考え方に寄っている。特に .form-inline、.form-group、.has-error は機械置換後に画面確認が必要。

7. コンポーネント変換仕様

領域Bootstrap 3系Bootstrap 5.3系変換仕様・注意点
Buttons.btn-default.btn-secondary などdefault色は廃止。画面設計上の意味でsecondary/light等を選択。
Buttons.btn-xs独自CSS または .btn-smxsサイズは標準なし。
Buttons.btn-block.d-grid で親を囲むgap utilityと組み合わせる。
Labels.label .label-primary.badge text-bg-primarylabelはbadgeへ統合。
Badges.badge.badge text-bg-secondary色指定はtext-bg-*またはbg-*へ。
Panels.panel .panel-heading .panel-body.card .card-header .card-bodypanelはcardへ置換。
Wells.well.card card-body または utilities余白・背景・枠線をutilityで再現。
Thumbnails.thumbnail.card または .img-thumbnail画像だけか、画像+説明かで選択。
Jumbotron.jumbotronutilitiesで独自ブロック化5系では標準コンポーネントなし。
Media object.media.d-flex utilitiesflex utilitiesで再現。
Glyphicons.glyphicon ...Bootstrap Icons等アイコンライブラリを別途導入。
Close button.close + &times;.btn-closeマークアップが変わる。
Navbar.navbar-default.navbar navbar-expand-* navbar-light bg-light色・展開ブレークポイントを明示。
Navbar.navbar-toggle.navbar-togglerdata属性もdata-bs-*へ。
Navbar.navbar-right.ms-auto右寄せはmargin-start autoへ。
Nav.active を .nav-item に付与.active を .nav-link に付与5系ではlink側に付ける。
Dropdown.dropdown-menu-right.dropdown-menu-endstart/end命名へ。
Modalsdata-dismiss="modal"data-bs-dismiss="modal"閉じるボタンのdata属性を変更。
Carousel.item.carousel-item項目class名を変更。
Carousel.left.carousel-control.carousel-control-prevprev/nextの構造へ変更。
Progress.progress-bar-success.progress-bar bg-success色はbackground utilityへ。

8. JavaScript・data属性変換仕様

対象Bootstrap 3系Bootstrap 5.3系変換仕様
data属性data-toggledata-bs-toggleすべてのBootstrap JS用属性にbs名前空間を付ける。
data属性data-targetdata-bs-targetcollapse/modal/tab/dropdown等で変更。
data属性data-dismissdata-bs-dismissmodal/alert/toast等で変更。
data属性data-parentdata-bs-parentaccordion/collapseで変更。
jQuery初期化$(el).modal("show")bootstrap.Modal.getOrCreateInstance(el).show()Bootstrap plugin呼び出しをVanilla JSへ。
イベント$("#m").on("shown.bs.modal", fn)el.addEventListener("shown.bs.modal", fn)イベント名は概ね維持、購読方法を変更。
Tooltip$("[data-toggle=tooltip]").tooltip()new bootstrap.Tooltip(el)querySelectorAllで複数初期化。
Dropdown$(".dropdown-toggle").dropdown()new bootstrap.Dropdown(el)data属性利用なら明示初期化不要な場合もある。

jQuery plugin呼び出しの変換例

// Bootstrap 3
$('#myModal').modal('show');
$('#myModal').on('shown.bs.modal', function () {{
  console.log('shown');
}});

// Bootstrap 5.3
const modalEl = document.getElementById('myModal');
const modal = bootstrap.Modal.getOrCreateInstance(modalEl);
modal.show();

modalEl.addEventListener('shown.bs.modal', () => {{
  console.log('shown');
}});

9. Sass / テーマ / 独自CSS変換仕様

対象Bootstrap 3系Bootstrap 5.3系変換仕様・注意点
言語LessSassBootstrap 5はSass。Less変数・mixinは再実装。
変数@brand-primary 等$primary 等命名・設計が大きく異なるためテーマ色として再マッピング。
カスタムCSSBootstrap 3のDOM構造前提Bootstrap 5のDOM構造前提panel/navbar/form等のDOM変更に合わせてセレクタ見直し。
色モード標準なしdata-bs-theme / CSS variables5.3ではdark modeや独自テーマを設計可能。
方向left/right前提start/end前提RTL対応のため物理方向指定を論理方向へ。
ビルドGrunt等の古い構成が多いVite/Webpack/Parcel/Sass等フロントビルド基盤も棚卸し対象。

10. 代表的な変換例

10.1 Panel → Card

<!-- Bootstrap 3 -->
<div class="panel panel-primary">
  <div class="panel-heading">タイトル</div>
  <div class="panel-body">本文</div>
</div>

<!-- Bootstrap 5.3 -->
<div class="card border-primary">
  <div class="card-header text-bg-primary">タイトル</div>
  <div class="card-body">本文</div>
</div>

10.2 Form group / validation

<!-- Bootstrap 3 -->
<div class="form-group has-error">
  <label class="control-label" for="email">メール</label>
  <input id="email" class="form-control" type="email">
  <span class="help-block">メールアドレスを入力してください。</span>
</div>

<!-- Bootstrap 5.3 -->
<div class="mb-3">
  <label class="form-label" for="email">メール</label>
  <input id="email" class="form-control is-invalid" type="email">
  <div class="invalid-feedback">メールアドレスを入力してください。</div>
</div>

10.3 Navbar

<!-- Bootstrap 3 -->
<nav class="navbar navbar-default">
  <div class="navbar-header">
    <button type="button" class="navbar-toggle collapsed" data-toggle="collapse" data-target="#nav">
      <span class="icon-bar"></span><span class="icon-bar"></span><span class="icon-bar"></span>
    </button>
    <a class="navbar-brand" href="#">Brand</a>
  </div>
  <div class="collapse navbar-collapse" id="nav">
    <ul class="nav navbar-nav navbar-right"><li class="active"><a href="#">Home</a></li></ul>
  </div>
</nav>

<!-- Bootstrap 5.3 -->
<nav class="navbar navbar-expand-lg navbar-light bg-light">
  <div class="container-fluid">
    <a class="navbar-brand" href="#">Brand</a>
    <button class="navbar-toggler" type="button" data-bs-toggle="collapse" data-bs-target="#nav" aria-controls="nav" aria-expanded="false" aria-label="Toggle navigation">
      <span class="navbar-toggler-icon"></span>
    </button>
    <div class="collapse navbar-collapse" id="nav">
      <ul class="navbar-nav ms-auto"><li class="nav-item"><a class="nav-link active" href="#">Home</a></li></ul>
    </div>
  </div>
</nav>

10.4 Modal trigger

<!-- Bootstrap 3 -->
<button class="btn btn-primary" data-toggle="modal" data-target="#myModal">開く</button>
<button type="button" class="close" data-dismiss="modal">&times;</button>

<!-- Bootstrap 5.3 -->
<button class="btn btn-primary" data-bs-toggle="modal" data-bs-target="#myModal">開く</button>
<button type="button" class="btn-close" data-bs-dismiss="modal" aria-label="Close"></button>

11. 変換後の確認観点

確認観点確認内容重点確認箇所
画面崩れ主要画面、共通部品、レスポンシブ幅で確認。グリッド、navbar、form、card置換部分。
JS動作modal、collapse、dropdown、tooltip、tab、carouselを確認。data-bs-*、Vanilla JS初期化。
フォーム入力、バリデーション、エラー表示、必須表示を確認。has-errorからis-invalidへの変更。
アクセシビリティaria属性、キーボード操作、フォーカス順を確認。collapse/modal/dropdown/navbar。
アイコンglyphicon置換後の表示・サイズ・色を確認。Bootstrap Icons等の依存追加。
IE依存IE専用CSS/JS、polyfill、条件付きコメントを削除または分離。5系はIE非対応。
独自CSSBootstrap 3のDOM・classに依存したセレクタを洗い出す。panel、navbar、form、dropdown周辺。
回帰試験スクリーンショット比較、E2E、入力系試験。単純置換では拾えない余白・折返し差分。

移行判定基準

  • Bootstrap 3のCSS/JSが残っていない。
  • jQuery plugin形式のBootstrap呼び出しが残っていない。
  • data-toggle、data-target、data-dismiss など旧data属性が残っていない。
  • panel、well、thumbnail、glyphicon など削除コンポーネントの置換方針が決まっている。
  • フォーム、ナビゲーション、モーダル、ドロップダウン、レスポンシブ表示について主要ブラウザで確認済み。
  • 独自CSSがBootstrap 5.3のDOM構造・class名に合わせて修正済み。
